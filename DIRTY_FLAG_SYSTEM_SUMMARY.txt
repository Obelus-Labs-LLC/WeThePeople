- BUT bad matches still present (Venezuela resolution for stock trading claim) ❌

ARCHITECTURE
============

Database Schema Change:
-----------------------
claims table:
  + needs_recompute INTEGER NOT NULL DEFAULT 0 (indexed)
    - 0 = False (evaluation is fresh)
    - 1 = True (evaluation needs recomputation)


Core Components:
----------------

1. utils/invalidation.py
   - invalidate_claims_for_bill(congress, bill_type, bill_number, db)
     Marks all claims matching this bill as needing recomputation
   
   - invalidate_all_claims(db)
     Nuclear option: mark ALL claims (for schema changes, etc.)
   
   - get_claims_needing_recompute(db, limit)
     Query claims with needs_recompute=1
   
   - clear_recompute_flag(claim_id, db)
     Clear flag after successful evaluation

2. jobs/enrich_bills.py (UPDATED)
   - After enriching a bill, automatically calls:
     invalidate_claims_for_bill(congress, bill_type, bill_number, db)
   - Returns claims_invalidated count in result

3. jobs/recompute_evaluations.py (UPDATED)
   - New parameter: dirty_only=False
   - When dirty_only=True, filters to needs_recompute=1
   - After successful evaluation, clears flag: claim.needs_recompute = 0
   - CLI support: --dirty-only flag


WORKFLOWS
=========

Automatic Invalidation on Enrichment:
-------------------------------------
1. Bill lifecycle data enriched (jobs/enrich_bills.py)
2. Find all ClaimEvaluations referencing this bill
3. Mark those claims with needs_recompute=1
4. Log: "Claims invalidated: N"

Example:
  python jobs/enrich_bills.py
  → Enriches HCONRES 68
  → Finds ClaimEvaluation for claim 1 (best_action_id → HCONRES 68)
  → Sets claim 1 needs_recompute=1


Efficient Recomputation:
------------------------
# Recompute ONLY dirty claims
python jobs/recompute_evaluations.py --dirty-only

# Recompute dirty claims for specific person
python jobs/recompute_evaluations.py --person-id=aoc --dirty-only

# Recompute dirty claims (max 100)
python jobs/recompute_evaluations.py --dirty-only --limit=100


On-Demand via CLI:

# Recompute specific claim
python -c "from jobs.recompute_evaluations import recompute_for_person; from models.database import SessionLocal, Claim; db = SessionLocal(); recompute_for_person(limit=1); db.close()"

   - Score: 1.000 (perfect match!)
   - Reason: Both have "Congress" keyword
   
   The enrichment added lifecycle context (progress_bucket, status_reason, timeline)

❌ NO RELEVANCE FILTERING
   - Matcher returns matches based on keyword overlap
