PHASE 4.1 DATA MODEL FIX: PersonBill Links (Stop Action Table Pollution)
=========================================================================

PROBLEM:
--------
The old ingestion system (ingest_robust.py) was inserting sponsorship/cosponsor
list items as Action evidence rows. This caused:
- Nonsensical counts (AOC showing 1646 "unique bills" in Action table)
- Table bloat (thousands of fake "actions" that aren't legislative actions)
- Confusion between relationship metadata and actual evidence

SOLUTION:
---------
Separate concerns into proper tables:
1. PersonBill: Link table for person↔bill relationships (Sponsored/Cosponsored)
2. Bill: Canonical bill table with needs_enrichment flag
3. BillAction: Bill lifecycle timeline (introduced, passed committee, etc.)
4. Action: ONLY for true evidence actions (votes, statements, etc.) - NOT sponsorship lists


FILES CHANGED:
--------------
1. models/database.py
   - Added PersonBill model (link table)
   - Added needs_enrichment flag to Bill model
   - Indexes on person_id, bill_id for efficient queries

2. ingest_robust_v2.py (NEW - replaces ingest_robust.py)
   - process_bill_link() creates PersonBill links + Bill stubs
   - Does NOT write to Action table (GUARDRAIL)
   - Marks bills with needs_enrichment=1
   - Full pagination + checkpointing preserved

3. verify_ingestion_v2.py (NEW - replaces verify_ingestion.py)
   - Reports PersonBill link counts (not Action counts)
   - Shows enrichment coverage (needs_enrichment flag)
   - Per-member sponsored/cosponsored breakdowns

4. cleanup_sponsorship_actions.py (NEW - Option B dedupe)
   - Identifies Action rows with "Sponsored bill:" or "Cosponsored bill:" markers
   - Safe deletion with backup export
   - Requires --confirm flag

5. test_personbill_links.py (NEW)
   - Verifies PersonBill links created
   - Verifies Action table NOT polluted
   - Verifies bills marked for enrichment

6. test_enrich_once.py (NEW)
   - Simulates bill enrichment
   - Verifies bill enriched once (even if linked to multiple people)
   - Verifies BillAction timeline created


DATABASE MIGRATION:
-------------------
New tables:
- person_bills (id, person_id, bill_id, relationship_type, source_url, created_at, updated_at)

Modified tables:
