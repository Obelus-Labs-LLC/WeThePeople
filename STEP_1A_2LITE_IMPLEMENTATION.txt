TASK 1: Link Extraction Fix
----------------------------
Problem: Schumer press releases were being ingested as "#aria-skip-press" (anchor links)
Root cause: Link extraction didn't strip URL fragments

Changes made to jobs/ingest_claims.py:
1. Updated extract_article_links() to:
   - Skip hrefs that start with "#" (anchor-only links)
   - Strip fragments from ALL URLs using .split('#')[0]
   - Added '#aria-skip' and '#skip' to skip_patterns
   - Normalize base URLs to include fragment stripping

Result:
✅ Schumer dry-run now produces 20 clean article URLs
✅ Zero fragment URLs (verified with verify_schumer_links.py)
✅ Example: "https://www.schumer.senate.gov/newsroom/press-releases/after-north-country-airports..."

TASK 2: Bill Reference Extraction (Step 2-lite)
------------------------------------------------
Added bill number extraction from full article text to capture bills not mentioned in extracted claim sentences.

Database changes:
1. Added claims.bill_refs_json column (TEXT, nullable)
   - Stores JSON array: ["H.R. 1234", "S. 5678"]
   - Migration: migrations/add_bill_refs_json.py (idempotent)

Code changes (jobs/ingest_claims.py):
1. New function: extract_bill_references(text)
   - Regex patterns for H.R. variants: H.R. 1234, HR 1234, H R 1234, H.R.1234
   - Regex patterns for S. variants: S. 1234, S 1234
   - Returns normalized sorted list: ["H.R. 1234", "S. 5678"]
   - Case insensitive

2. Updated process_article() to:
   - Extract bill refs from full article text
   - Store as JSON in claim.bill_refs_json
   - Applied to ALL claims from same article

Model changes:
- Updated models/database.py Claim model to include bill_refs_json column
